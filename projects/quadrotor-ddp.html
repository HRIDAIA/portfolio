Quadrotor 3D DDP Guidance Simulation

This project follows the AE4803 Robotic Systems & Autonomy final project brief from Georgia Tech (Fall 2022). The assignment tasks a 12-state quadrotor with flying from (-3, -2, -1) m to (5, 3, 2) m over an 8 s horizon using differential dynamic programming (DDP) with Δt = 0.01 s, while exploring solver regularization, line search, and barrier-state obstacle avoidance.

The write-up will capture how I extend the provided MATLAB specification—covering recursive MPC, safety envelopes, and experimental validation artifacts—into a modern toolchain.

Spec Highlights I’m Implementing

  • State vector x = (x, y, z, ˙x, ˙y, ˙z, ϕ, θ, ψ, p, q, r) with body rates mapped to Euler rates through the standard 1-2-3 rotation matrix.
  • Quadrotor parameters: 0.5 kg mass, I = diag(0.0032, 0.0032, 0.0055) kg·m², 0.17 m moment arm, kt = 0.01691/m torque constant, and thrust inputs u = (u₁…u₄).
  • Translational equations mẍ = R[0,0,u₁+…+u₄]ᵀ + [0,0,−mg]ᵀ with full Euler-angle rotation coupling, plus matching rotational dynamics for p, q, r.
  • Obstacle field enforced via discrete barrier states: h₁(x) = (x−2.2)² + (y−2.2)² + (z−1)² − 1, h₂(x) = x² + (y+0.2)² + z² − 1, and h₃(x) = (x−3)² + y² + (z−0.5)² − 1.

Upcoming Breakdown

  • Document DDP implementation details—trust-region regularization, line search scheduling, and cost-weight tuning for the point-to-point maneuver.
  • Show how I wrap the same model in a recursive MPC loop to compare solver reuse, runtime, and constraint handling.
  • Visualize barrier-state enforcement with 3D trajectory plots, per-axis time histories, and control effort comparisons.
  • Optional bonus: port the policy to Robotarium unicycles to demonstrate feedforward versus feedback executions.
